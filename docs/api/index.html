    API

    Integration APIs

    Jun 7, 2022

    This page explains how to create use AireFrame’s apis to build custom integrations

    Authentication

    All integration APIs require a bearer token provided by the OAuth client credentials flow - this means that they cannot be called on behalf of a user.

    The bearer token must have a tenant_key claim.

    Unsolicited Inbound Subject API

    AireFrame can handle unsolicited subject create/update messages provided that the unsolicited inbound configuration has been set for the tenant see here.

    Endpoint: https://aireframe.com/api/v1/messaging/subject

    Body:

    {
    "metadata": {
        "key": value
    },
    "customFieldValues": {
        "key": "value"
    }
}

    Where:

    • metadata is an optional dictionary of values that are passed to downstream systems listening to AireFrame’s notifications e.g. AireFlow
    • customFieldValues is a required dictionary of tenant custom field keys and the subject values, all required fields must be provided

    The response will be the created subject:

    {
    "internalId": "d6df60bb-0c6d-47ab-855b-23c98d58ee79",
    "externalId": "b38a64fe-df82-4510-92bb-314e26ff5509",
    "customFieldValues": [
        {
            "customField": {
                "id": "0eabc586-c18a-4ff5-8883-081e1615ea72"
            },
            "value": "value"
        }
    ]
}

    Where internalId is AireFrame’s subject identifier

    Create Subject Form API

    Endpoint: https://aireframe.com/api/v1/messaging/form

    Body:

    {
    "metadata": {
        "key": value
    },
    "subjectId": guid,
    "key": string,
    "version": integer (optional),
    "correlationId": string (optional),
    "userId": string (optional)
}

    Where:

    • metadata is an optional dictionary of values that are passed to downstream systems listening to AireFrame’s notifications e.g. AireFlow
    • subjectId is AireFrame’s internal identifier for the subject (guid)
    • key is the key for the form that is to be created
    • version is the optional version of the form that is to be created, if omitted it will default to the latest version of the form
    • correlationId is an optional identifier to group together forms and events
    • userId is an optional identifier of the user who should be shown the created form if they are currently active

    The response will be the created form:

    {
    "instanceId": "56ba2e87-32db-423f-ab11-997cc81b3dfb",
    "subjectId": "f0a8a0ec-6eba-43d9-bcf9-029315876b59",
    "createdAt": "2022-08-10T14:45:06.54816Z",
    "viewLink": "http://localhost:8081/patient/f0a8a0ec-6eba-43d9-bcf9-029315876b59/instance/56ba2e87-32db-423f-ab11-997cc81b3dfb/view",
    "editLink": "http://localhost:8081/patient/f0a8a0ec-6eba-43d9-bcf9-029315876b59/instance/56ba2e87-32db-423f-ab11-997cc81b3dfb/edit"
}
