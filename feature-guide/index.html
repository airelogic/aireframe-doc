What is AireFrame? ​

AireFrame is an intuitive, easily integrated, low code, case management solution which enables data to be recorded against a ‘subject’ (this may be a patient, citizen or other).

As well as capturing data, AireFrame offers powerful reporting and visualisation tools, allowing users to create custom reports and dashboards to meet their needs. There is also out-of-the-box support for a self-access portal, allowing subjects to securely access their own data.

It offers deep integration with the other AireSuite products, including AireFlow - our workflow and task management solution, and AireForms - our data capture solution.

It can run as a standalone system where all data is captured within the system and stored in the cloud, integrated with external systems using the SDK or the AireGlu data provider, or a combination of both.

Concepts ​

There are a number of key concepts in AireFrame that will be referred to throughout the documentation:

  • Tenancy - A tenancy (tenant-environment) is a single instance of AireFrame. It is a self-contained environment that can be configured as the user requires.
    • For more information on tenants, see the AireIdentity guide.
  • Subject - A subject is the entity that you are recording data about. This could be a patient, a citizen, a case, or any other entity that you need to record information about.
    • Note, only a single subject type can be configured per tenant.
  • Data Source - Data sources are where subject-related data is stored. We provide an internal data source where data is stored in the cloud, but you can also configure AireFrame to connect to external data sources.
  • Subject Provider - The subject provider is the data source that provides the subjects and the organisation structure data to AireFrame.
