    Transfer Configuration

    Exporting and importing a tenants configuration

    Apr 11, 2023

    This page details how to export and import a configuration into AireFrame. This allows you to easily transfer your configuration across multiple tenants.

    What can you find on this page:

    • Required Permissions
    • Setup Git Repository
    • Export Configuration
    • Import Configuration

    Required Permissions

    • To setup the git repository used for import/export you will need the View Git Repository and Set Git Repository permissions.
    • In order for you to export a configuration you will need the Export Configuration permission.
    • For you to import a configuration you will need the Import Configuration permission.

    Setup Git Repository

    In order to use the import/export functionality you will need to setup a git repository. This is where the configuration files will be written to and read from.

    The git repository settings use AireVault variables and secrets so the configuration can be shared across products.

    1. Go to the Transfer Configuration screen: Configuration -> Transfer Configuration
    2. Select the Git Repository tab from the top bar. You should see a screen as follows
    3. Select the relevant AireVault variables and secrets for the options.
      • Note when using SSH authentication, the uri must be in the format ssh://[user]@[host][:port?]/[path] e.g. ssh://git@github.com/my/repo"
    4. Save the settings, this will attempt to connect to the repository
      • If the connection is successful, the settings will be saved and you will be able to export and import configurations
      • If the connection is unsuccessful, you will be presented with an error message and the settings will not be saved

    Export Configuration

    In order to export a configuration, a git repository must be configured.

    Exporting a configuration will generate a JSON file of the current configuration and write to the git repository as the latest version. If there are no existing versions, this will be version 1.

    1. Go to the Transfer Configuration screen: Configuration -> Transfer
    2. Click the export button
    3. A JSON file containing the configuration will be written to the git repository as the latest version.

    Import Configuration

    In order to import a configuration, a git repository must be configured.

    Importing a configuration will update any items that matches the key and type and create any that do not exist. This can be a risky endeavour so if any of the imports fail it will restore back to the latest version of the configuration.

    After clicking confirm the import job is run in the background as it may take some time. You will be notified on completion or failure.

    1. Go to the Transfer Configuration screen: Configuration -> Transfer
    2. Select the Import tab from the top bar.
    3. A list of the available versions will be displayed. Select the version you would like to import.
    4. Click the Import button
    5. A warning will remind you that this could cause complications where you can either cancel or continue with the import
