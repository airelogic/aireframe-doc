    Dashboards

    Jun 19, 2022

    This page explains how to create dashboards in aireframe. For example you may want to show different widgets on the home or subject pages.

    What you can find on this page:

    • Required Permissions
    • Create Dashboard
    • Delete Dashboard

    Required Permissions

    • In order to create dashboards you will the following permissions:
      • Create Dashboard
      • List Dashboard
      • View Dashboard
    • In order to edit a dashboard you will need the following permissions:
      • Edit Dashboard

    Create Dashboard

    1. Go to the dashboards configuration screen: Configuration -> Data Visualisations -> Dashboards (Tab).
    2. Click add new. You will see the following screen
    3. Enter a title and key for your dashboard.
    4. Decide if the dashboard should be used on the portal site, only dashboards marked as ‘for porta;’ will be available to add to the subject self-access section.
      • Only widgets marked as ‘for portal’ will be available to add to this dashboard.
    5. Drag over the widgets you want to enable from the Add Widgets section to the widgets input.
    6. Click save changes.

    You can also use this page to edit dashboards.

    Delete Dashboard

    1. Go to the dashboard configuration screen: Configuration -> Data Visualisations -> Dashboards (Tab).
    2. Select the dashboard you wish to delete.
    3. Click remove dashboard.

    Section Configuration

    Once you have created a dashboard you need to configure which section(s) it should be shown on. You can do this by clicking on the ‘Sections’ tab on the configure visualisations page.

    This page also allows you to set the default dashboard that should be shown on each section. Note, this is only used if the user has not selected a dashboard to show.

    Options: In AireFrame, there are currently five sections where dashboards can be shown:

    • Location - This is where you see the widgets on the structural entity (location) overview page. In this section, the widgets are populated with data for all subjects located within that structural entity. Note that this is recursive - it shows data for subjects that are located in child structural entites and so on
    • Subject List - This is where you see the data shown on the subject list alongside their id and name. In this section, the visualisation is slightly altered to retrieve only the first data point for that subject (according to the default ordering)
    • Subject Record - This is where you see the widgets on the subject page with data only for that subject
    • Subject Preview - This is where the subject information is shown in the right hand side as a preview, as well as any widgets that are configured to show in this section.
    • Subject Portal - This is what a subject will see when they log in to the portal.
