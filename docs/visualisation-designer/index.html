    Create custom visualisations

    Apr 3, 2023

    This page explains how to create custom visualisations in aireframe. For example, you may want to show structured data in a table or graph.

    What you can find on this page:

    • Required Permissions
    • Visualisation Builder
    • Enabling/Disabling Visualisations

    Required Permissions

    • In order to create visualisations you will the following permissions:
      • List Visualisation
      • Create Visualisation
      • View Data Source
    • In order to edit visualisations you will need the following permissions:
      • List Visualisation
      • View Visualisation
      • Edit Visualisation
      • View Data Source

    Create Visualisation

    Go to the visualisation configuration screen (Configuration -> Visualisation) then click the plus button in the top right of the table.

    Visualisation Builder

    The builder is split into three stages:

    1. Define the data pipeline
    2. Design the visualisation
    3. Add metadata

    Define the data pipeline

    In this stage you select a data extract and then optionally define any transformations you want to take place on the dataset. See the guide on how to configure data extracts here.

    The possible transformations are:

    • Filters
      • These are boolean liquid expressions that remove any data points which evaluate to false
      • If more than one filter is defined, these are applied as union (i.e all filters must evaluate to true for the data point to be included)
    • Mappers
      • These are liquid expressions that add a new field to each data point
      • Mapping expressions cannot reference other mapped fields
      • The output datatype is inferred from the expression, however if the expression contains more than one statement it will always evaluate as a string
      • The output key must be unique amongst the initial and mapped field
      • Unless, case/when and loops are not currently supported
    • Aggregators
      • These are operators that aggregate the data points into a single value
      • Aggregators can operate over one or more fields (provided the data types are valid for that aggregate method)
      • If an aggregator can return more than one value (e.g. count and mode) only one aggregator is allowed
      • The output key must be unique amongst the aggregated fields

    If the pipeline is valid, the output of the pipeline will be displayed as a preview.

    It is only possible to continue to the next step once the data pipeline is valid.

    Design the Visualisation

    First, choose how you want to display the data (The options will depend on the output of the pipeline).

    For a table:

    1. Drag and drop the columns you wish to display on the table
    2. The columns can be reordered by dragging them to a new position. (Top-bottom = Left-Right)
    3. The columns can be removed by clicking the bin icon within each series.
    4. Customise each column by changing the display name and selecting a formatter (if applicable)

    For a pie chart:

    1. Select the series you want to display on the pie chart
    2. Customise the series by changing the display name and selecting a formatter (if applicable)

    For a graph:

    1. Drag and drop the series you wish to use as the X-Axis.
      • Only one series can be selected as the x-axis, to change remove the series and add a new one
    2. Drag and drop the series you wish to use display on the Y-Axis.
      • Multiple series can be displayed on the y-axis
      • Each series on the y-axis must have a chart type selected
    3. Customise the series by changing the display name and selecting a formatter (if applicable)

    Once you are happy with your visualisation click next to continue.

    Add metadata

    1. Enter a title for your visualisation.
    2. Select which series should be used to order the visualisation by default
      • For charts this will most likely be the series on the x-axis
    3. Select the default order direction

    Once you are happy with the configuration of your visualisation, click save.

    Enabling/Disabling Visualisations

    Once you create a visualisation you are able to add it to a collection to enable it for the view that you wish to use it on. See Collection Configurator.
