    Subject Configuration

    Configure the subject name, custom and computed fields

    Dec 7, 2021

    This page explains how to change the subject configuration in AireFrame. For example you may want to change the name of the subject or add/remove custom fields.

    What you can find on this page:

    • Required Permissions
    • Configure Subject Provider

    Required Permissions

    • In order to change the subject configuration you will need the following permissions:
      • View Subject Configuration
      • Edit Subject Configuration

    Change Subject Configuration

    Go to the subject configuration screen: Configuration -> Subject Configuration.

    Here you can:

    • Change the subject name
    • Change the subject provider
    • Change the properties of the custom fields
    • Remove a custom field
    • Add a custom field - drag new fields from the left hand side into the relevant section
    • Re-order the subject banner - drag and drop fields to re-arrange

    Subject banner grouping

    On this page you can re-order the custom fields to alter how they are shown on the subject banner.

    • Primary field - A single field that is shown at the top of the banner, even when minimised
    • Secondary fields - These fields are shown when the banner is expanded, in the specified order (top to bottom)
    • Hidden fields - These are input fields that are not shown on the subject banner but still appear in subject forms. E.g. You can capture information about a subject using several input fields; then use a computed field to join these together and hide the induvidual fields.

    Dropdown lists

    Dropdown lists allow you to add your own custom options. This is useful when there are a set number of answers which are very long, or where it will useful to pull the data straight into a histogram or pie chart without needing any additional processing (e.g. natural language processing).

    To create a dropdown list:

    • Drag the ‘Dropdown List’ control from the list of available fields
    • Give the field a name, such as Nationality, Ethnicity, or even Favourite Colour
    • Click the plus symbol [ + ] next to ‘Add Value’ for as many options as you need
    • Write the text for each option
    • Use the cross [ x ] next to each option to remove any which are unnecessary

    Computed Fields

    Computed fields are flexible and allow you to reformat or combine the custom fields from the table above. This is achieved by using the liquid syntax.

    To use the custom fields as variables in the liquid mapping of a computed field, use the value from the ‘key’ column in the corresponding row above. Please note, the field keys are automatically generated from the field names and must be unique.

    Formatting

    We allow certain HTML tags to be rendered on the page. Currently these are: <b />, <strong />, <em />, <i />, <small />, <sub />, <sup />, <br />.

    It is possible to use these tags in both the computed fields and the actual subject data. However, we reccomend only using it on the computed fields since downstream systems may be not be expecting HTML tags.
